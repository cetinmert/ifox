  • VMP signing service

    Since content protection (widevine / DRM) and licensing are very difficult, castlabs has brought a new solution to this situation with the EVS signature method.
    Sites that require content protection licenses such as Netflix, Hulu, Prime, Spotify now work smoothly.

    # The EVS client is published as a Python 3.7+ module, castlabs-evs, and can be installed/upgraded from PyPi with:
    py -m pip install --upgrade castlabs-evs
    Make sure you have Python installed on your computer and install this module.

    # To create an EVS account you would execute something like this:
    py -m castlabs_evs.account signup
    You must be registered with EVS to sign. Please enter the information correctly and completely.

    # Signing an ECS application package
    py -m castlabs_evs.vmp sign-pkg node_modules\electron\dist
    The signing method works only in npm start, i.e., in test mode. Follow the guide below for packaging.

    https://github.com/castlabs/electron-releases/wiki/EVS

    Code Signing for Windows

    Navigate to the project folder and open the terminal (Console, CMD).
    - First, build the project: npm run build-win
    - Next, run the following command to sign the package: py -m castlabs_evs.vmp sign-pkg dist/win-unpacked
    - Go to the dist/win-unpacked folder and copy the projecName.exe.sig (.exe.sig) file.
    - Open the "sig" folder in the project's root directory and paste the file into it.
    - To ensure that signing is effective across all platforms (.setup, .appx, .exe), we need to repackage.
    - Rebuild: npm run build-win
    - That's it. To test, open Netflix, Hulu, Prime, Spotify, or any other application and check if it is working correctly.
